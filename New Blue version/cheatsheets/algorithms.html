Algorithms

Simple Sorting

Insertion Sort
Fast for a very small number of elements. Classicly bad otherwise.

Example Objects: raw (raw list), and sorted, initially [].

Let sorted[0] = raw[1]. For each element in raw[1:length(raw)], loop through sorted[n:1] until sorted[i] > element. Insert element into sorted[i+1].

  • Worst Case Run Time: an^2 + bn + c
    • Merge Sort
    Faster than Simple Sorting, especially for larger sequences.

    Merge sort uses the divide and conquer strategy:

    Divide: Divide your n-elements into two subsequences of n/2 elements each.

    Conquer: Sort each subsequence recursively using merge sort. (split up again into 2 parts, until you only have 2 elements per group, and then sort).

    Combine: Merge the two sorted subsequences to produce the sorted answer.

    Two sorted sets of elements can merged into one sorted set quite quickly. Imagine two decks of sorted cards (so that the top card is the smallest), each with n/2 cards, facing up. Simply pick the smaller card and place that card into a final pile. Repeat. At worst, this takes n operatoins.
